## 2. 核心玩法详细设计

### 2.1 基础循环
角色从左侧出发 → 自动匀速向右移动（卷轴滚动）
↓
音符/收集物从右侧生成，向左移动
↓
玩家在判定区（屏幕中部偏右）点击屏幕
↓
根据时序精度判定 Perfect / Great / Good / Miss
↓
击中 → 收集物被吸收 + 特效 + Combo 计数
Miss → 无惩罚，收集物飞过
↓
歌曲结束 → 结算评分（S/A/B/C）+ 最大 Combo


### 2.2 判定窗口

| 评级 | 窗口 | 得分倍率 |
|------|------|----------|
| Perfect | ±50ms | ×1.0 |
| Great | ±100ms | ×0.8 |
| Good | ±150ms | ×0.5 |
| Miss | >150ms | 0，断 Combo |

### 2.3 Combo 系统

- 连续击中：Combo +1
- Miss 或 Good：Combo 归零
- 分数加成：`基础分 × (1 + Combo × 0.05)`
- 结算时展示 **最大 Combo 数**

### 2.4 结算评级

| 评级 | 条件 |
|------|------|
| S | 总分 ≥ 95% 理论满分，Miss ≤ 5 |
| A | 总分 ≥ 85% |
| B | 总分 ≥ 70% |
| C | 总分 < 70% |

### 5.3 🎯 MIDI 转时间线工作流（核心）

**你的 MIDI 文件就是收集物的“出现时间表”。** 这一套流程是游戏可玩性的基石。

#### 工作流
你的 MIDI 文件
↓
tools/midi_extract.py（Python 脚本）
↓
config/{歌曲}.json（节拍时间线）
↓
游戏加载 JSON → 驱动收集物在准确时间点生成

#### MIDI 准备规范

在 DAW 中（如 FL Studio / Cubase / 免费的 MuseScore）:

1. 导入歌曲音频
2. 新建一轨，选择任意乐器（如钢琴）
3. **在需要玩家点击的位置写入音符**（统一使用 MIDI 音符编号 72，即 C5，方便脚本识别）
4. 导出为 MIDI 文件（仅这一轨）

#### 提取脚本功能

`tools/midi_extract.py` 需要实现：

- 读取 MIDI 文件
- 找到所有音符编号为 72 的音符
- 提取每个音符的 **起始时间（秒）**
- 输出 JSON 数组，格式如下

#### 输出 JSON 格式（完整示例）

```json
{
  "songId": "levan_polkka",
  "title": "甩葱歌",
  "artist": "初音未来",
  "bpm": 240,
  "audioSrc": "assets/audio/levan_polkka.mp3",
  "offset": -0.05,
  "duration": 100,
  "notes": [
    { "time": 1.23, "type": "normal", "lane": 1 },
    { "time": 1.56, "type": "golden", "lane": 2 },
    { "time": 2.00, "type": "special", "lane": 1, "value": "ユ" }
  ],
  "backgroundShiftPoints": [
    { "time": 30, "bgType": "chorus" },
    { "time": 60, "bgType": "verse" }
  ],
  "characterSwitchPoints": [
    { "time": 35, "character": "yanhe" }
  ]
}

关键字段说明

time - 音符出现的绝对秒数（驱动收集物生成）

type - normal/golden/special，控制收集物外观和分值

offset - 全局偏移量（秒），用于微调整体延迟

人工精修
MIDI 导出后必须人工校对，在文本编辑器中直接调整各 time 数值，直到游戏内手感完美。这是最耗时但最重要的一步。

8. 技术要点
8.1 音频同步（最重要）
Javascript
// ✅ 使用 AudioContext.currentTime
const audioContext = new AudioContext();
const elapsed = audioContext.currentTime - startTime;
const nearestNote = findNearestUnprocessedNote(elapsed);
// ❌ 不要用 setTimeout/setInterval

核心原理
音乐播放的进度 不能 用 setTimeout / setInterval——它们有几十毫秒的偏差，音游判定完全没法用。

必须使用 Web Audio API 的 AudioContext.currentTime，这是一个硬件级的精确定时器，误差 < 1ms。

第 1 步：创建 AudioContext 并加载音频
Javascript
// 全局 AudioContext（整个游戏只创建一个）
const audioContext = new AudioContext();
// 用 fetch 加载音频文件，解码为 AudioBuffer
async function loadAudio(url) {
  const response = await fetch(url);
  const arrayBuffer = await response.arrayBuffer();
  const audioBuffer = await audioContext.decodeAudioData(arrayBuffer);
  return audioBuffer;
}
第 2 步：播放音频，记录精确开始时间
Javascript
let startTime = 0;  // AudioContext 时间轴上的起点
let audioBuffer = null;
async function startSong(buffer) {
  audioBuffer = buffer;
  
  // 创建音源
  const source = audioContext.createBufferSource();
  source.buffer = audioBuffer;
  source.connect(audioContext.destination);
  
  // 记录播放开始的精确时间（AudioContext 的高精度时钟）
  startTime = audioContext.currentTime;
  
  // 开始播放（可以加 offset 调整同步）
  source.start(0);
  
  // 注意：必须在用户交互事件中调用，否则浏览器会阻止播放
  // 例如 touchstart / click 回调里调用此函数
}
第 3 步：在游戏循环中获取当前播放进度
Javascript
function getCurrentSongTime() {
  return audioContext.currentTime - startTime;
}
// 游戏循环（requestAnimationFrame）
function gameLoop() {
  const currentTime = getCurrentSongTime(); // 精确当前秒数
  
  // 检查所有未处理的音符
  checkNotes(currentTime);
  
  // 渲染收集物、角色等...
  render();
  
  requestAnimationFrame(gameLoop);
}
第 4 步：判定玩家点击
Javascript
// 音符数据（从 JSON 加载）
const notes = [
  { time: 1.23, type: 'normal', processed: false },
  { time: 1.56, type: 'golden', processed: false },
  // ...
];
// 玩家点击时调用
function handlePlayerTap() {
  const clickTime = getCurrentSongTime(); // 精确时间
  
  // 找到最近的一个未处理音符
  let nearestNote = null;
  let minDiff = Infinity;
  
  for (const note of notes) {
    if (note.processed) continue;
    const diff = Math.abs(clickTime - note.time);
    if (diff < minDiff && diff < 0.2) { // 只看 200ms 以内的音符
      minDiff = diff;
      nearestNote = note;
    }
  }
  
  if (!nearestNote) return; // 没有可命中的音符
  
  // 判定评级
  const diff = Math.abs(clickTime - nearestNote.time) * 1000; // 转为毫秒
  
  let rating;
  if (diff <= 50)      rating = 'Perfect';
  else if (diff <= 100) rating = 'Great';
  else if (diff <= 150) rating = 'Good';
  else                  rating = 'Miss';
  
  // 标记为已处理，避免重复判定
  nearestNote.processed = true;
  
  // 触发得分、特效等
  onNoteHit(nearestNote, rating);
}

避坑要点
易错点	正确做法
用 new Audio().currentTime	❌ 精度不够，与渲染循环不同步
用 setInterval(checkNotes, 16)	❌ 误差累积，判定不准
直接用 audioContext.currentTime 而不记录 startTime	✅ 必须记录播放起点，相减得到歌曲进度
不处理 AudioContext 的暂停状态	必须在用户手势中调用 audioContext.resume()
